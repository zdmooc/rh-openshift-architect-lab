# Checklist EX280 – Points à maîtriser

Coche chaque point quand tu te sens à l’aise **sans regarder la doc**.

- [ ] Projets / namespaces : création, suppression, changement de projet.
- [ ] Utilisateurs / rôles : `oc adm policy` pour donner/retirer des droits.
- [ ] Deployments / ReplicaSets / Pods : création, mise à jour, rollback.
- [ ] Services / Routes : exposition interne/externe.
- [ ] ConfigMaps / Secrets : création, montage, mise à jour.
- [ ] PV / PVC / StorageClass : création et utilisation dans les pods.
- [ ] Probes (readiness / liveness) : ajout et diagnostic.
- [ ] Ressources (requests/limits) : configuration et impact.
- [ ] Logs / events : `oc logs`, `oc describe`, `oc get events`.
- [ ] Sauvegarde/export de manifests : `oc get -o yaml > fichier`.
