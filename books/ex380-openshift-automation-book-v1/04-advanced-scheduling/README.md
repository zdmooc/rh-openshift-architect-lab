
# 04 – Scheduling avancé

Objectifs :

- Utiliser taints/tolerations pour contrôler le placement des pods.
- Protéger la disponibilité via `PodDisruptionBudget`.
- Utiliser (anti-)affinity et éventuellement `topologySpreadConstraints`.
- Gérer les défauts de projet (tolerations, LimitRange, ResourceQuota).
